# Visual Grammar Guide

## 画面の大きさ

- **全景** (`wide`): 場所全体と人物・物の関係を一度に見せる。 塔や駅など、まず状況を理解させたい時。
- **中景** (`medium`): 人物や対象と周辺の状況を一緒に読ませる。 行動と背景の意味を同時に見せたい時。
- **寄り** (`close`): 手がかりや表情など一つの対象へ注意を集める。 見落としてほしくない情報がある時。
- **極端な寄り** (`extreme_close`): 小さな刻印や異常を画面いっぱいにする。 重要な細部を一瞬で記憶させたい時。
- **図版全画面** (`graphic_fullframe`): 台帳、仮説、選択肢を図版として読ませる。 情報の比較や文字の意味が映像より重要な時。

## カメラの動き

- **固定** (`locked`): 画面を動かさず事実や違和感を落ち着いて見せる。 例: 空の鐘枠を静止して見せる。
- **ゆっくり寄る** (`slow_push`): 一つの手がかりへ注意を集める。 例: 真鍮の蛾の刻印へ近づく。
- **ゆっくり引く** (`slow_pull`): 距離を取り未解決感や余韻を残す。 例: 最後に空の鐘枠から離れる。
- **横へ振る** (`slow_pan`): 同じ空間で視線を次の対象へ移す。 例: 台帳の行から消えた名前へ移る。
- **奥行き差を動かす** (`controlled_parallax`): 前景と背景を少しずらし静かな奥行きをつくる。 例: 評議会の影と台帳を別の層で動かす。
- **図版を溶かしてつなぐ** (`graphic_dissolve`): 図版を重ねながら切り替え連想をつなぐ。 例: 複数の仮説札を答えにせず入れ替える。

英語 enum は機械 Blueprint との照合用で、日本語説明が制作上の入口です。
