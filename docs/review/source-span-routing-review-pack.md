# Source-Span Routing Review Pack

## What This Pack Adds

`fff-source-span-routing-review-pack-001` turns the existing deterministic adapter expansion outputs into a focused supervision packet. It does not change extraction behavior. It gives the reviewer one place to inspect whether the current fixture spans and routing are useful enough before freeform review drives more adapter work.

The primary UI is still `public/review/index.html`. In the current `fff-review-procedure-lock-001` Review Hub, open the `Source Audit` / `抽出根拠レビュー` mode to reach this pack without scanning the full story review and project cockpit layers. The full machine-readable pack is `artifacts/source-span-routing-review-pack.json`.

This pack preserves `fff-model-api-boundary-spec-001`; it does not replace the boundary spec or start provider integration. It inserts a source-span/routing supervision step before deciding whether model/API work should proceed.

## Inputs

| Source | Path | Role |
| --- | --- | --- |
| Fixture memos | `artifacts/extraction-adapter-fixtures/` | Local raw memo sources. |

| Pack generator | `tools/fff-source-span-review-pack.mjs` | Builds the review pack and validates each extraction output while reading it. |

## Cross-Fixture Result

The generated pack records:

- 3 fixture memos.
- 36 extracted elements.
- 27 profile candidates.
- 20 claim candidates.
- 12 timeline candidates.
- 0 source-span mismatches.
- 0 missing source refs.
- 0 unsafe visual routing cases.
- 0 non-held review defaults.
- 0 human-owned decision adopt suggestions.
- 17 elements guarded by human-owned review boundaries.

This means the automated readback is clean enough for human quality review. It does not mean the spans are artistically ideal, the extraction choices are final, or any candidate is canon.

## Review Surface

The Review Hub shows six representative elements per fixture:

- A person route.
- A human-owned object route.
- A visual route.
- A placeholder or rule-gap route.
- An unresolved decision route.
- A source reference route.

Each representative row shows the extracted value, raw source snippet, span locator, routing target, confidence/default status, human-owned guard, and review notes. The JSON pack keeps all 36 rows for full readback.

## Review Debt

The pack separates automated guard failures from human review debt:

| Debt class | Current automated result | Next review question |
| --- | --- | --- |
| Weak span | 0 mismatches | Does the span capture enough source context without becoming too broad? |
| Over-broad span | Not automatically detected | Should any phrase be narrowed before model/API work? |
| Vague extraction | Not automatically detected | Does the extracted value preserve the memo wording? |
| Routing ambiguous | 0 unsafe visual routes | Should any route move between Claim, Timeline, Profile, Visual, or Human Review? |
| Default too confident | 0 non-held defaults and 0 human-owned adopt suggestions | Do the held defaults still match the risk level? |
| Fixture class missing | 6 intentional gaps | Add only the fixture classes that real review feedback needs. |

Fixture classes still visible as future candidates after the multilingual expansion: very broad source spans, translated memo text, and model/API provider envelope output. Contradictory claims, malformed/missing source-span payloads, sparse bullet-only notes, and multilingual memo text now have separate local readbacks.

## Command

```powershell
node .\tools\fff-source-span-review-pack.mjs .\artifacts\extraction-adapter-fixtures .\artifacts\extraction-adapter-outputs .\artifacts\local-extraction-adapter-expansion-smoke-result.json .\artifacts\source-span-routing-review-pack.json
```

The command validates every adapter output through `tools/fff-state.mjs validate-extraction` while building the pack.

## Boundaries

This slice does not add model/API behavior, provider credentials, database persistence, publishing adapters, AI video generation, production sync, or final canon decisions. Toma fate, brass moth truth, and Council motive remain human-owned unresolved decisions.

## Next Recommended Continuation

Use freeform review against the Review Hub section or the JSON pack. If feedback names a concrete span/routing issue, revise the deterministic fixture output or add a narrow fixture. If feedback only asks for broader coverage, add one source-span fixture class at a time before designing a model/API adapter behind the same validator boundary.
